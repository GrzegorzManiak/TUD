Tips for visual design.

I have compiled and interpreted some general rules of user interface design according to design teams, industry leaders and professors such as Ben Shneiderman, I've added some interactive examples to push my points across.

Here they are in a non-sequential order.

Consistency

The user interface should be consistent with the rest of the application, take a look at Spotify as an example as their app is on multiple devices.

As you can see, the app is very consistent, with the same general design on all devices.

Shortcuts

Again, taking a look at Spotify, you can see that there are a lot of shortcuts, such as the 'search' shortcut (Ctrl-l), the 'next track' shortcut (Ctrl-right), and the 'previous track' shortcut (Ctrl-left).

Feedback

An example of Feedback is the common folder, once a user hovers over one, it becomes highlighted, once they click it, a border is shown around the folder, indicating that the user has that folder selected.

We can expand this by changing the folder look depending if it has files in it or not, if you take a look at "Folder 2", you can tell that it contains files.

Dialogue

We need to inform the user that something has happened or is happening, like downloading a file, updating their profile, etc. We must notify our users, to not leave them confused.

let us use this as our example, a user is installing a program and they have run out of space, mid installation.

As the user performs actions that might take time to execute, the user will be prompted with a dialogue box, the user wants to know exactly what's happening, you want to be as descriptive as possible.

Clarity

Be clear, don't be cryptic, if your user is creating an account, and e.g. their password does not match the minimum requirements, tell them what's missing.

Ease of use

To make a great website, you want to make the users visit as friction-free as possible, lets take a look at this as our example.

We need the user to input their four-digit 2FA code, this is a very nominal task and should be made as easy to do as possible. In the first example, I demonstrate a proper implementation, there are four boxes, clearly signifying that the user needs to input the four digits they just received into their according boxes, each interaction provides visual feedback to the user.

In the second example, I demonstrate an improper implementation, the user needs to input 4 digits, but there's just one input field, not specifying the input length, there's no feedback provided at all, leaving the user in the dark.
